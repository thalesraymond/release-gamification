/opsx.propose Title: Mobile Release API Motivation: Currently, the system tracks mobile releases internally via webhooks and stores them in the database, but there is no way for clients to view or retrieve these releases. Providing a read-only API is the natural next step to bridge the gap between backend data collection and frontend consumption. Specification: Add ListMobileReleases and GetMobileRelease use cases. Expose GET /mobile-releases and GET /mobile-releases/:id Fastify API endpoints. Enhance MongoMobileReleaseRepository with findAll and findById methods, using MongoDB projections (.project({ _id: 0 })) for optimal performance.
